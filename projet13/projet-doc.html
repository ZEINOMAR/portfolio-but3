Documentation Technique

Objectif du projet

Mettre en place un système de gouvernance des données industrielles pour améliorer l'accès et la compréhension des données issues des capteurs industriels (cuves d'électrolyse) au sein du Laboratoire de Recherche et Fabrication (LRF) de Rio Tinto.

Technologies utilisées

  • PostgreSQL
  • Docker & Docker Compose
  • DataHub (catalogage de données)
  • Grafana + Volkov Labs Business Forms
  • Raspberry Pi OS
  • Dockge
  • JavaScript / SQL

Travaux réalisés

  • Analyse comparative des solutions open-source de data catalog
  • Installation et configuration de DataHub en Docker
  • Création de domaines, tags, propriétés structurées, requêtes SQL types
  • Création d'interfaces de saisie de données dans Grafana (insertion & update SQL)
  • Migration de certains services Docker sur Raspberry Pi 4B via Dockge
  • Rédaction de la documentation technique et formation de l'équipe métier

Exemple de commandes utilisées

sudo docker-compose up -d
docker ps
git clone https://github.com/datahub-project/datahub
sudo apt install postgresql
psql -U postgres -d bddlrf

Problématiques rencontrées & solutions

  • Incompatibilité Docker ARM64 → migration sur machine Linux classique.
  • Restrictions de sécurité réseau → déploiement local hors réseau Rio Tinto.
  • Dépendance métier aux experts → documentation centralisée des tables et schémas.
  • Difficulté utilisateurs sur SQL → formulaires interactifs avec Grafana Business Forms.
